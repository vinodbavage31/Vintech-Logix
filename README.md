</div>

| Technology | Purpose | Why We Chose It |

| ![HTML5](https://img.shields.io/badge/HTML5-e34f26?style=flat-square&logo=html5&logoColor=white) | Structure & Content | Semantic, accessible, SEO-friendly |
| ![CSS3](https://img.shields.io/badge/CSS3-1572b6?style=flat-square&logo=css3&logoColor=white) | Styling & Animations | Lightweight, no framework bloat |
| ![Google Fonts](https://img.shields.io/badge/Google%20Fonts-4285f4?style=flat-square&logo=google&logoColor=white) | Typography | Beautiful, web-optimised typefaces |

### Why Pure HTML & CSS?

Most developers reach for heavy frameworks immediately. We made a conscious, strategic choice:

```
  Zero JavaScript dependencies    →  Faster load times
  Pure semantic HTML              →  Maximum SEO benefit
  No framework bloat              →  Leaner codebase
  CSS animations & transitions    →  Still visually rich
  Universal browser compatibility →  Works everywhere
  Easier community contribution   →  Lower barrier to entry
```

> *"The fastest website is one that loads exactly what it needs — nothing more, nothing less."*

## 🔀 How to Fork & Submit a PR

</div>

**Step 1 — Fork the Repository**

Click the **Fork** button at the top-right corner of this page.

---

**Step 2 — Clone Your Fork Locally**

```bash
git clone https://github.com/YOUR-USERNAME/Vintech-Logix.git
cd Vintech-Logix
```

---

**Step 3 — Create a New Branch**

> Always work on a dedicated branch — never commit directly to `main`.

```bash
git checkout -b feature/your-feature-name

# Examples:
# git checkout -b feature/dark-mode
# git checkout -b fix/navbar-mobile-alignment
# git checkout -b docs/improve-readme
```

---

**Step 4 — Make Your Changes**

Open the project in your favourite editor (VS Code recommended) and get building.

```bash
code .
```

---

**Step 5 — Stage & Commit Your Changes**

```bash
git add .
git commit -m "feat: add dark mode toggle to navbar"

# Good commit message examples:
# feat: add hero section animation
# fix: mobile menu overflow issue
# docs: update README contributing guide
# style: improve button hover effects
```

---

**Step 6 — Push to Your Fork**

```bash
git push origin feature/your-feature-name
```

---

**Step 7 — Open a Pull Request**

1. Go to your forked repo on GitHub
2. Click **"Compare & pull request"**
3. Write a clear title and description of what you changed and why
4. Tag it with relevant labels — `bug`, `enhancement`, `documentation`, etc.
5. Submit and wait for review!

---

### PR Checklist

Before submitting, please verify:

- [ ] Code is clean and properly formatted
- [ ] No unnecessary files included — no `.DS_Store`, `node_modules`, etc.
- [ ] Changes tested in a browser
- [ ] Commit messages are clear and descriptive
- [ ] Pull request targets the `main` branch
